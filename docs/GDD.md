## 1. 한 줄 소개 (엘리베이터 피치)
> 이 게임을 한 문장으로. "당신은 ___에서 ___을 하는 ___ 게임."

**당신은 친구와 함께 마을에서 장비를 만들고, 챕터를 헤쳐 나가 어려운 보스의 패턴을 학습해 쓰러뜨리는 2D 픽셀 협동 액션 게임.**

## 2. 비전 / 핵심 재미
> 플레이어가 느껴야 할 핵심 감정·경험은? 왜 이 게임인가?

- **핵심 재미 = 어려운 보스를 (파티로) 학습해서 잡는 재미.** 처음엔 죽고, 패턴을 읽고, 결국 잡는다 — 몬스터헌터·마비노기 영웅전 계열의 전투 그 자체의 재미.
- 전투는 **조작과 판단**으로 이긴다. 뱀서류식 빌드 발사대(자동 공격을 구경하는 게임)가 아니다.
- 빌드업은 있다: 재료를 모아 장비를 만들고 강화해 다음 도전을 준비하는 리듬이 전투를 받친다. 단, 성장은 전투를 돕는 수단이지 전투를 대신하지 않는다.
- 🔒 **튜닝 원칙(확정): 장비는 입장권이 아니라 단축키.** 장비가 없어도 도전은 가능하게 조절한다 — 장비는 실수 허용치를 늘려줄 뿐, 문을 여는 게 아니다.
- 멀티(2인 협동)에서 재미가 극대화되지만 솔로로도 완주 가능하다.

## 3. 장르 · 시점 · 플랫폼
- **장르:** 2D 협동 보스전 액션 (수렵 액션 계열 — 마을 제작 + 챕터 사냥 루프)
- **시점:** **탑다운 — 확정**
- **아트 스타일:** 픽셀아트(도트) — 확정
- **플랫폼:** 웹 (브라우저 / HTML5·WASM 익스포트) — 확정
- **엔진:** Godot 4.7.1 (렌더러 Compatibility — 웹 필수)
- **멀티:** 솔로 지원 + 멀티 권장. 해커톤 빌드는 **2인 협동 기준** (최대 4인은 이후 확장). 연결 = **WebSocket 중계 서버(무료 티어) — 확정.** 초대 = **방 코드** (여유 시 코드 포함 링크 공유 추가).
- **저장:** 계정/서버 저장 없음 — **브라우저 로컬 저장.** 저장 시점 = **스테이지 클리어마다.**

> ⚠ **동기화 주의:** 이 절의 "플랫폼(웹)·엔진·렌더러"는 `CLAUDE.md` 상단 「정체」 줄과 **같은 사실을 미러**한다. 한쪽을 고치면 반드시 다른 쪽도 같이 고친다 — 어긋나면 `projectb-critic`이 잡는다.

## 4. 핵심 루프 (Core Loop)
> 플레이어가 반복하는 30초~몇 분짜리 사이클.

**마을에서 제작·강화한다 → 챕터에 나가 스테이지를 돌파하며 재료를 얻는다 → 보스를 잡는다(패턴 학습) → 마을로 복귀해 더 좋은 장비를 만든다 → 다음 챕터/재도전.**

- **마을:** 걸어 다니는 씬. 제작·강화·창고·파티 준비의 거점.
- **챕터:** 여러 스테이지를 **한 호흡에 연속 진행**. 스테이지 사이에 **모닥불**(체크포인트).
- **챕터 스케일 흐름:** 중간 스테이지들(잔몹·엘리트, 조작 학습·패턴 암시) → **최종 스테이지 = 보스.** 보스는 챕터당 1마리, 최종 스테이지에만 있다.
- **전멸 룰:** 파티 전원 사망(솔로에선 본인 사망) 시 **마을로 귀환.** **재도전은 챕터 처음부터 — 확정** (스테이지 스킵·보스 직행 없음).
  - ⚠ **설계 제약:** 재도전 1회 비용 = 챕터 재주파이므로, **중간 스테이지는 짧게** 유지해야 한다. 챕터당 스테이지 수·길이 결정(§11)은 이 제약 아래에서.
- **루프를 닫는 힘:** 보스가 어려워서 죽는다 → 장비 제작/강화로 재도전 여력을 만든다 → 그러나 승부는 결국 패턴 학습으로 낸다.

## 5. 핵심 메커니즘 (Mechanics)
> 이동·전투·상호작용 등 플레이어가 실제로 하는 행동.

- **조작:** **키보드 + 마우스 — v1.6 전부 확정.** **이동 = WASD · 조준 = 마우스 방향 · 기본 공격 = 마우스 좌클릭 · 스킬 = 우클릭 + Q/E/R · 구르기(회피) = Shift · 상호작용 = F.** 몸통 스프라이트는 **2방향(좌/우 플립)** — 조준선은 마우스로 360°지만 캐릭터 그림은 좌우만 뒤집는다(아트 비용 최소). **구르기에는 i-frame(구르는 동안 무적)이 있다 — v1.6 확정** (웹 멀티 지연에서도 "분명히 굴렀는데 맞음" 억울함이 없도록; 기믹 위주 패턴 원칙과 짝).
- **전투:** 마우스로 조준해 공격을 내지르고, 보스 패턴을 보고 **구르기로 피하고** 때린다. 판정·타이밍이 승부처. (판정 반경·데미지 등 수치는 사용자가 플레이하며 조인다 — 확정하지 않음.)
- 🔒 **패턴 설계 원칙(확정): 기믹 위주.** 보스 패턴은 긴 텔레그래프·예고 장판 등 **읽고 대응하는 기믹** 중심으로 설계한다. 프레임 단위 반응을 요구하는 패턴 금지 — 웹 멀티 지연에서도 공정해야 한다.
- **직업 — 궁수·전사·법사 3종.** 시작 시 선택, 이후 고정. **직업 = 플레이 방식의 변환** (사거리·리듬·역할이 다름). **개발은 전사부터 완성 — 확정.** 상세 스킬 구성은 TBD.
- **스테이지 클리어 조건 (중간 스테이지):** 모든 몬스터 처치 **또는** 엘리트 처치. 상자방(전투 없는 보상방)도 고려.
- **데스 룰:** 스테이지 내에서 죽으면 **그 스테이지 동안 부활 불가(관전)**. 파티가 스테이지를 클리어하면 **HP 1로 부활.** 파티 전원 사망 = 전멸 → 마을 귀환(챕터 처음부터). **솔로 사망 = 전멸과 동일.**
- **모닥불:** 스테이지 사이 체크포인트. **각자 자리에 앉아야 하고, 앉아 있는 동안 천천히 회복 — 확정.** (앉기 = 시간 비용이자 무방비 연출; HP1 부활자의 회복 지점 역할도 겸한다)
- **드랍:** **공유 드랍 — 확정.** 적 처치 시 드랍 1벌이 생성되고 **아무나 먼저 줍는다(선착).** 롤·픽업 선착은 **호스트 권한**으로 확정한다. (개인별 드랍이 아닌 이유 = 2인 협동 + 아이템 이전 자유라 개인별 이점이 미미하고, 네트워크·개발이 크게 단순해짐 — 해커톤 "간단한 개발" 대전제.) 아이템 이전은 여전히 자유 — 바닥에 떨구거나 창고를 경유. **바닥에 떨군 아이템은 누구나 주울 수 있고, 스테이지를 넘어가면 사라진다 — 확정.**
- **드랍 4종 — 확정:** ① **골드**(강화 비용) ② **일반 재료** ③ **핵심 재료** ④ **도면(설계도)**. **완성 장비는 드랍하지 않는다 — 장비는 오직 제작으로 얻는다.** 런(run) 한정 소모품·버프는 비범위(§10).
- **인원 스케일링:** **솔로일 때 보스가 약해진다 — 확정.** (조정 수치·범위는 TBD → §11)

## 6. 진행 · 성장 (Progression)
> 무엇으로 강해지고 무엇을 잠금 해제하는가? 세션 간 무엇이 남는가?

- **역할 분담:** **직업 = 플레이 변환**(어떻게 싸우는가), **장비 = 수치 변환**(얼마나 세게/단단하게).
- 🔒 **해커톤 범위에서 장비는 수치만 — 확정.** 공격 모션 변경 등 플레이를 바꾸는 장비는 이후 확장(§10 비범위).
- **강화:** 장비를 강화해 **수치가 오른다. 강화 비용 = 골드 — 확정.** (강화 확률·단계·실패 유무 상세는 TBD → §11)
- **재료 — 출처 확정:** **핵심 재료 = 보스**(제작용). **일반 재료 + 골드 = 잔몹·엘리트 드랍.** 스테이지 클리어 시 저장되어 **전멸해도 마지막 클리어분은 남는다** — "죽고→강화→재도전" 루프의 연료(§4 루프·§11 제약 충족).
- **장비 슬롯·스탯 골격 — 확정:** 장비 부위 = **무기 1 + 방어구 1(2슬롯).** 강화 스탯 2종 = **공격력(무기)·체력(방어구).** (구체 밸런스 수치는 사용자가 플레이하며 조인다 — 확정 안 함.)
- **도면(설계도) 시스템 — 확정:** 도면 = **레시피 언락 토큰.** 드랍 시 "설계도 획득!" 연출로 발견의 쾌감(즉시) → 마을에서 재료로 제작해 장비를 얻음(지연 보상). **도면은 한 번 얻으면 영속**(로컬 저장). **완성 장비는 직접 드랍하지 않는다 — 장비 획득 경로는 오직 제작.** 출처 = **보스는 핵심 장비 도면 확정, 엘리트·상자방은 보조 도면 확률.**
- **세트효과:** **시스템 틀만 잡는다.** 해커톤 콘텐츠로는 채우지 않는다 (§10 비범위).
- **메타 진행:** 브라우저 로컬 저장(스테이지 클리어마다) — 장비·재료·진행도가 세션 간 남는다. **멀티 방의 챕터 선택은 방장의 해금 기준 — 확정.**
- **2차 전직:** 추후 예정 — 해커톤 비범위 (§10).

## 7. 콘텐츠 (Content)
> 적·아이템·레벨·스테이지 등. 데이터로 확장 (→ projectb-rules §4 "새 X = 파일 한 장").

- **월드:** 마을 1개(걸어 다니는 씬) + 챕터 (해커톤 확정 = 챕터1, 스트레치 = 챕터2).
- **챕터 구성:** 중간 스테이지 여러 개(잔몹/엘리트, 상자방 고려) + 사이 모닥불 + **최종 스테이지 = 보스 1.** (챕터당 스테이지 수 TBD — §4 "짧게" 제약 아래)
- **적 3계층:** 잔몹(조작 익히기용) / 유닉·엘리트(보스 패턴 암시) / 보스(핵심 콘텐츠, 챕터당 1). 보스의 구체 패턴·이름은 TBD.
- **아이템:** **무기 1 + 방어구 1(2슬롯 — 확정)**, 강화·제작 재료(골드·일반 재료·핵심 재료), **도면**(레시피 언락). 개수는 챕터1 완주에 필요한 최소로.
- **드랍 배치:** 잔몹·엘리트 = 골드 + 일반 재료(엘리트·상자방 = 보조 도면 확률), 보스 = 핵심 재료 + 핵심 장비 도면. **완성 장비 드랍 없음 — 장비는 제작으로만.**
- **온보딩:** 직업 선택 → 튜토리얼 제작템 1개 → 마을 제작 튜토리얼 → 챕터1 진입.

## 11. 열린 질문 (Open Questions)
> 아직 정하지 못한 결정들. 여기 쌓아두고 하나씩 닫는다.

- **재접속 처리 — 리드 권고, 확정 대기:** 끊긴 사람만 그 판에서 제외(마을로), 남은 사람은 계속 진행(다음 스테이지부터 솔로 스케일 적용), 재합류는 다음 판부터. 판 도중 재합류 동기화는 비범위.
- **부채꼴 시야 제한** (탐색 구간 한정) — 채택/보류/폐기 미정.
- **인원 스케일링 수치:** 솔로 시 보스 약화는 확정 — 무엇을 얼마나 낮추는가(체력만? 패턴도?).
- **강화 상세:** 강화 단계·성공 확률·실패 유무. (스탯 = 공격력·체력 2종, 장비 부위 = 무기·방어구 2슬롯, 비용 = 골드는 §6에서 확정.)
- **창고 시스템 상세:** 용량·접근 위치·공유 여부.
- **상자방 상세:** 등장 빈도·보상 종류.
- **모닥불 상세:** 회복량·속도, 앉아 있는 동안 피격 가능 여부.
- 챕터당 스테이지 수(§4 "짧게" 제약)·길이, 보스 패턴 상세, 직업별 스킬 구성, 기준 해상도·캐릭터 크기.
- 손맛 수치(판정 반경·데미지 밸런스 등)는 확정하지 않는다 — **사용자가 플레이하며 조인다.**
