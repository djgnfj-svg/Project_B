# Project_B 작업 리스트 (구현 순서표)

> **이 파일이 구현 순서의 정본이다.**
> 기획이 "무엇인가"는 `docs/GDD.md`, 구현 규율은 `CLAUDE.md` — 이 파일은 "무엇을 어떤 순서로 만들지"만 다룬다.

## 운영 규칙

1. **사용자**가 「작업 큐」에 구현할 항목을 순서대로 적는다.
2. **클로드(리드)** 는 위에서 아래 순서로만 구현한다 — 임의로 순서를 바꾸지 않는다.
3. 항목 하나가 끝나면 체크박스를 채우고(`[x]`) 결과물 경로·비고를 기록한다.
4. **작업 턴이 끝날 때마다 클로드는 이 파일을 다시 읽고 상태를 갱신한다.**
5. 항목이 모호하면 구현 전에 사용자에게 확인한다.

## 확정된 마을 디자인 결정 (2026-07-23)

- **레이아웃 = E안 계열** (docs/mockups/village_mockup_e.html): 남쪽 바위 기슭 강 + 나무다리 + 낚시 데크(기능 없음, 분위기), 북쪽 기능 건물 3채, 우측 돌계단+출발 게이트. 최종 승인 대기.
- **집은 외관만 — 내부 씬 없음.** 문 앞에 NPC가 서 있고 말을 걸면 패널이 열린다 (NPC는 표시 전용 → 멀티 동기화 불필요).
- **NPC 구성 (확정):** 기능 = 촌장(광장 모닥불 옆, 지팡이, 길라잡이) · 대장장이(제작소, 근육남) · 강화술사(강화소 — "준비중") · 창고지기(상자 퍼리 — F로 보관함). 분위기 = **낚시꾼(데크에 앉아 낚싯대+찌, 밀짚모자)** · **문지기(게이트 옆, 창)** · **토끼(벚꽃나무 아래)**. 전부 표시 전용(동기화 불필요).
- **숲 테두리는 성기게** — 위쪽 한 겹(간격 넓게)+옆 조각 몇 그루만. 마을을 빽빽하게 두르면 갇힌 느낌(사용자 피드백) — 실제 씬에서도 여백을 남긴다.
- **건물 외형은 기능이 읽히게 (레퍼런스 확정):**
  - 제작소 = `_refs/ref_craft_house.jpg` 기준 — 짙은 슬레이트 지붕 + 돌 굴뚝(연기) + **밖으로 드러난 발광 화덕 셰드** + 앞마당 모루(돌 받침)·잉걸불 통·무기 진열대
  - 강화소 = `_refs/ref_enhance_house.jpg` (던파 키리의 실린더) 기준 — **흰 벽 + 파란 지붕** + 건물 옆 **발광 유리 강화 실린더 기계**(브라스 캡·기포) + 검은 파이프 보일러 + "준비중" 팻말
  - **창고 = 건물 없음** — 대형 궤짝(철 밴드·자물쇠) + 상자 더미가 곧 창고. **상자 퍼리 창고지기에게 F(상호작용) → 보관함(인벤토리형) 패널**이 열린다. 창고지기 = 살아있는 상자(귀·볼터치·짧은 발).
- **벚꽃나무 = 마을 중앙 포인트 컬러** — 광장 옆 큰 것 1 + 서쪽 작은 것 1 (C안에서 채택).
- 🔴 **자연물 랜덤성 규칙:** 나무·바위·덤불은 스프라이트 변형 2~3종 제작 + 배치 시 flip_h·스케일(±15%)·톤 지터를 준다 — 같은 스프라이트 반복 배치 금지 (사용자 피드백: "나무가 딱딱하다").
- 레퍼런스: `_refs/ref_river.jpg`(마을 80% 기준) · `_refs/ref_village.jpg`(밝은 카툰 20%) · `_refs/ref_blacksmith_craftpix.webp`(대장간 소품 문법) · `_refs/ref_blacksmith_props.png`(대장간 외관 구조)
